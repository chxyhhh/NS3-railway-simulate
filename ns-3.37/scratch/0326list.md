  D. 运行命令

  cd F:/bishe/NS3-railway-simulate/ns-3.37

  # 编译
  ./ns3 build

  # baseline（都关）
  ./ns3 run "tcp_high_speed_train --expCase=baseline"

  # 仅 rwnd 收缩
  ./ns3 run "tcp_high_speed_train --expCase=rwnd-only --triggerSource=position"

  # 仅 ACK 拆分
  ./ns3 run "tcp_high_speed_train --expCase=ack-only --triggerSource=position"

  # 联合优化（默认）
  ./ns3 run "tcp_high_speed_train --expCase=joint --triggerSource=position"

  # 时间触发模式
  ./ns3 run "tcp_high_speed_train --expCase=joint --triggerSource=time --triggerTimesSec=5.5,17.5,29.5"

  # 自定义参数
  ./ns3 run "tcp_high_speed_train --expCase=joint --triggerAdvanceMs=300 --ackSplitCount=4 --ackSplitK=0.6
  --rwndAlphaFloor=0.25 --rwndBeta=3.0"

  E. 编译风险点与自检清单

  1. build/include/ns3/tcp-socket-base.h 需同步 — ns-3 的 build 目录下有头文件副本，编译时 cmake
  应自动更新；若未更新需手动 ./ns3 configure 后再 build
  2. 静态成员链接 — 14 个 static 成员在 .cc 中定义，确保编译时 internet 模块重新编译
  3. lambda 捕获 — Simulator::Schedule 中的 lambda 使用无捕获 []() 访问静态成员，ns-3.37 支持 C++17 应无问题
  4. simTime 替换 — Simulator::Stop(Seconds(60.0)) 改为 Seconds(simTime)，默认 simTime=30 需通过命令行调整或修改默认值
  5. 触发位置默认值 — eNB 间距 1000m，默认触发位置 450,1450,... 对应各 handover 边界前 ~50m，可根据实际 handover
  时刻微调
